# DevDAO 기여 가이드

## 개요
DevDAO는 백야 프로토콜의 기술 개발을 담당하는 핵심 DAO입니다. 여기서는 코드 기여를 통해 프로토콜의 발전에 참여하고 B 토큰을 획득할 수 있습니다.

## DCA (Decentralized Contribution Activity) 목록

### 1. Pull Request - 250B
**검증기준**: Closed(merged)

**활동 내용**:
- 백야 프로토콜 관련 코드 개선, 새로운 기능 추가, 버그 수정 등의 Pull Request 제출
- 기존 코드의 성능 최적화 또는 리팩토링
- 새로운 모듈이나 컴포넌트 개발

**검증 과정**:
1. GitHub에서 백야 프로토콜 저장소에 Pull Request 제출
2. 코드 리뷰 과정을 거쳐 승인 받기
3. Pull Request가 메인 브랜치에 병합(merge)되면 자동 검증 완료
4. 병합 후 자동으로 250B 지급

**기여 팁**:
- 명확한 커밋 메시지와 PR 설명 작성
- 코드 스타일 가이드 준수
- 충분한 테스트 코드 포함
- 기존 코드와의 호환성 고려

### 2. Pull Request Review - 120B
**검증기준**: Closed(merged)

**활동 내용**:
- 다른 개발자의 Pull Request에 대한 코드 리뷰
- 코드 품질 검토 및 개선 제안
- 버그 발견 및 보안 이슈 지적
- 최적화 방안 제시

**검증 과정**:
1. 다른 개발자의 Pull Request에 의미 있는 리뷰 코멘트 작성
2. 승인(Approve) 또는 변경 요청(Request Changes) 제출
3. 해당 Pull Request가 최종적으로 병합되면 자동 검증 완료
4. 병합 후 자동으로 120B 지급

**리뷰 품질 기준**:
- 코드 로직의 정확성 검토
- 성능 및 보안 관점에서의 평가
- 코드 스타일 및 구조 개선 제안
- 테스트 코드의 충분성 검토
- 건설적이고 구체적인 피드백 제공

### 3. Issue Report - 80B
**검증기준**: Closed(merged)

**활동 내용**:
- 버그 리포트 작성
- 새로운 기능 제안
- 개선사항 제안
- 사용자 경험 관련 이슈 제기

**검증 과정**:
1. GitHub Issues에 문제점 또는 제안사항 상세히 기록
2. 재현 가능한 버그의 경우 재현 단계 명시
3. 해당 이슈가 해결되어 닫히면 자동 검증 완료
4. 이슈 해결 후 자동으로 80B 지급

**이슈 작성 가이드**:
- 명확하고 구체적인 제목
- 문제 상황의 자세한 설명
- 버그의 경우 재현 단계 포함
- 예상 결과와 실제 결과 비교
- 스크린샷이나 로그 파일 첨부 (필요시)
- 제안사항의 경우 구체적인 구현 방안 제시

## GitHub 연동 설정

### 1. GitHub 계정 연동
1. DevDAO 참여하기 모달에서 "연동하기" 버튼 클릭
2. GitHub 계정으로 로그인
3. 백야 프로토콜 저장소 접근 권한 허용
4. 자동 검증 시스템 활성화

### 2. 자동 검증 시스템
- **실시간 모니터링**: GitHub API를 통해 실시간으로 활동 추적
- **자동 B 토큰 지급**: 검증 기준 달성 시 즉시 B 토큰 지급
- **중복 방지**: 동일한 기여에 대한 중복 보상 방지 시스템
- **투명성**: 모든 기여 활동과 보상 내역 공개

## 개발 환경 설정

### 필수 도구
- **Node.js** (v16 이상)
- **Git**
- **코드 에디터** (VS Code 권장)

### 저장소 클론
```bash
git clone https://github.com/baekya-protocol/baekya-protocol.git
cd baekya-protocol
npm install
```

### 개발 서버 실행
```bash
npm run dev
```

### 테스트 실행
```bash
npm test
```

## 코딩 스타일 가이드

### JavaScript
- ES6+ 문법 사용
- 세미콜론 필수
- 2스페이스 들여쓰기
- camelCase 변수명 사용
- 의미 있는 변수명과 함수명

### 커밋 메시지 컨벤션
```
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 스타일 변경
refactor: 코드 리팩토링
test: 테스트 코드 추가/수정
chore: 빌드 프로세스 또는 보조 도구 변경
```

## 기여 시 주의사항

### 보안
- 개인 키나 비밀번호 등 민감한 정보 코드에 포함 금지
- 보안 취약점 발견 시 공개 이슈 대신 maintainer에게 직접 연락
- 제3자 라이브러리 사용 시 보안 검토 필수

### 품질
- 모든 새로운 코드에 대한 테스트 코드 작성
- 기존 테스트가 통과하는지 확인
- 코드 리뷰 피드백 적극 수용
- 문서화 (README, 코드 주석 등) 충실히 작성

### 커뮤니티
- 존중하고 건설적인 소통
- 다양한 관점과 의견 수용
- 초보자도 이해할 수 있는 명확한 설명
- 오픈소스 정신과 가치 존중

## 기여 프로세스

### 1. 이슈 확인
- 기존 이슈 중 작업할 만한 것 선택
- 새로운 기능 제안 시 먼저 이슈로 논의

### 2. 브랜치 생성
```bash
git checkout -b feature/your-feature-name
```

### 3. 개발 및 테스트
- 기능 구현
- 테스트 코드 작성
- 로컬 테스트 실행

### 4. Pull Request 제출
- 명확한 제목과 설명
- 관련 이슈 링크
- 변경사항 요약
- 테스트 결과 포함

### 5. 코드 리뷰
- 리뷰어의 피드백 수용
- 필요시 코드 수정
- 승인 후 병합

## 보상 시스템

### B 토큰 지급 시점
- Pull Request 병합 시: 즉시 지급
- Pull Request Review 완료 시: 해당 PR 병합 시 지급
- Issue 해결 시: 이슈 클로즈 시 지급

### 추가 인센티브
- 월간 최고 기여자: 보너스 B 토큰
- 연간 우수 기여자: 특별 NFT 발급
- 커뮤니티 추천을 받은 기여: 추가 보상

---

DevDAO는 백야 프로토콜의 미래를 함께 만들어갈 개발자들을 환영합니다. 여러분의 기여가 새로운 탈중앙화 사회 시스템의 기반이 됩니다.
